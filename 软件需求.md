自动化耳机测试与数据上传软件需求文档

1. 引言

1.1 目的
本文档旨在明确“自动化耳机测试与数据上传软件”（以下简称“本软件”）的功能需求、性能要求、界面要求及交互流程，为软件开发、测试及验收提供依据。

1.2 适用范围
本软件适用于耳机生产线的质量检测、研发阶段的性能验证以及实验室数据采集。用户通过本软件连接音频测试硬件，自动完成耳机声学性能测试，并将测试结果与产品信息上传至指定网站服务器。

1.3 术语与缩写
| 术语 | 解释 |
|------|------|
| DUT | 被测设备（耳机） |
| SPL | 声压级 |
| FFT | 快速傅里叶变换 |
| API | 应用程序编程接口 |
| 频响 | 频率响应 |
| THD | 总谐波失真 |
| 动态线性偏离 | 不同输出电平下响应的线性偏差 |

2. 项目概述
2.1 项目背景
当前耳机测试多依赖人工操作，效率低且易引入人为误差。为提高测试一致性与数据可追溯性，需开发一套自动化测试软件，实现从产品信息录入、自动校准、多项声学参数测试到数据上传的全流程自动化。

2.2 目标
- 实现一键式自动化测试，无需人工干预音量调节。
- 支持频响、失真、相位、冲击响应、动态线性偏离五项核心指标测试。
- 测试前强制录入产品信息（品牌、型号、单元配置、定价）并上传产品照片。
- 测试结果自动与产品信息绑定，并通过网络上传至企业网站数据库。

3. 总体业务流程
1. 用户启动软件，连接测试硬件（声卡、测量麦克风、耳机）。
2. 用户填写产品信息（品牌、型号、单元配置、定价）并上传至少一张产品照片。
3. 用户将耳机放置于测试夹具中，点击“开始测试”。
4. 软件自动播放1kHz正弦波，调节输出音量使测量麦克风处声压级达到94dB SPL（±0.1dB）。
5. 依次执行以下测试项目并记录数据：
- 频率响应（20Hz – 20kHz，正弦扫频）
- 总谐波失真（THD，20Hz – 20kHz）
- 相位响应（20Hz – 20kHz）
- 冲击响应（脉冲信号，测量瞬态响应）
- 动态线性偏离：分别以94dB基准、94dB+10dB、94dB-10dB三个电平播放扫频信号，计算三次频响的差值（高电平响应－低电平响应），评价线性度。
6. 测试完成后，软件自动生成测试报告（数据文件），并将产品信息、照片（或照片URL）、测试数据打包，通过HTTP请求上传至指定网站。
7. 上传成功后，软件提示完成并进入下一只耳机的测试流程。

4. 功能需求

4.1 硬件接口与配置
- 音频设备管理：自动检测并选择输入/输出设备（支持ASIO/WASAPI/MME），支持设置采样率（至少44.1kHz/48kHz/96kHz可选）、缓冲区大小。
- 通道配置：指定播放通道（驱动耳机）和录音通道（连接测量麦克风），支持左右声道独立或同时测量。
- 校准文件导入：支持加载测量麦克风的校准文件（如频率响应补偿文件），以修正测试结果。
- 输出电平校准：软件内置数字音量控制，可精确调整播放电平，配合测量麦克风实现94dB SPL自动对准。
4.2 产品信息录入
- 表单输入：提供文本输入框，必须填写以下字段：
- 品牌（字符串）
- 型号（字符串）
- 属性（字符串）
- 单元配置（如动圈/动铁/圈铁，单元数量等，字符串）
- 定价（数值，支持货币符号或纯数字）
- 照片上传：支持从本地选择图片文件（jpg/png），可预览，至少上传一张，可多张。照片将随测试数据一起上传。
4.3 自动测试流程

4.3.1 音量自动对准
- 播放1kHz正弦波，连续监测测量麦克风采集的声压级。
- 软件自动调整播放增益，直至声压级稳定在94dB ±0.5dB。
- 若无法达到目标（如耳机灵敏度过低或过高），应给出明确提示并中止测试。

4.3.2 频响测试
- 采用对数扫频信号（20Hz – 20kHz），或稳态正弦步进（96PPO）。
- 记录每个频率点的声压级，生成频响曲线数据。

4.3.3 失真测试
- 同频响测试的激励信号，对录音信号进行FFT分析，计算每个频率点的THD（%）或THD+N。
- 记录各频率点失真值，生成失真曲线。

4.3.4 相位测试
- 基于扫频信号，测量过剩相位。
4.3.5 冲击响应
- 保存冲击响应文件。
4.3.6 动态线性偏离
- 在94dB基准电平下完成频响测试后，分别将输出电平提升10dB和降低10dB（保持播放增益调整，不改变硬件设置），再次进行频响测试。
- 计算：高电平频响（dB）－低电平频响（dB），得到偏离曲线，反映不同声压级下的线性度。
- 可同时输出高、中、低三条频响曲线用于对比。

4.4 数据管理与上传
- 本地存储：测试结果应保存为结构化文件（如JSON/CSV），包含产品信息、测试时间、测试数据、照片路径（或嵌入）。
- 上传功能：调用网站提供的HTTP API（RESTful），以multipart/form-data或JSON格式提交数据，包括：
- 产品信息字段
- 照片文件（或上传后返回的URL）
- 测试数据文件（或JSON编码的数据）
- 上传状态反馈：显示上传进度，成功或失败提示；失败时可重试或手动保存本地。
5. 非功能需求
5.1 性能要求
- 单次完整测试时间（不含人工操作）不超过20秒。
- 音量自动对准时间不超过5秒。
- 软件界面操作响应及时，无卡顿。

5.2 可靠性
- 测试过程中如果硬件断开或发生异常，应自动停止并提示用户。
- 关键步骤（如音量校准）应具备超时保护（例如30秒内未达标则报错）。

5.3 兼容性
- 操作系统：Windows 10/11 64位。
- 音频硬件：支持任何标准Windows音频接口（ASIO驱动优先）。
- 网站API：HTTPS协议，支持JSON格式。

5.4 易用性
- 界面简洁，流程引导清晰，一键开始测试。
- 提供实时声压级监控仪表和结果预览，帮助用户定位耳机放置是否正确。
5.5 可扩展性
- 测试项目配置化，便于后续增加新测试项。
6. 界面原型要求
6.1 主窗口布局
- 顶部：标题栏，显示软件名称及当前版本。
- 左侧/顶部面板：产品信息录入区（表单+照片选择）。
- 中央：实时监控区（声压级表、波形图）、测试进度条。
- 右侧/底部：开始测试按钮、停止按钮、日志信息显示区。
- 状态栏：硬件连接状态、网络状态、当前测试状态。

6.2 测试结果预览（可选）
- 测试完成后，可弹出简要结果窗口，显示关键指标（如平均失真、频响平坦度）并询问是否上传。

7. 接口定义

7.1 硬件接口
- 音频输出：播放测试信号至耳机（左右声道可独立/同时）。
- 音频输入：接收测量麦克风信号，用于声学测量。

7.2 软件接口
- 上传API需由网站端提供，例如：
- URL：`https://api.example.com/upload_test`
- 方法：POST
- 参数：
- `brand` (string)
- `model` (string)
- `driver_config` (string)
- `price` (number)
- `photos` (file[])
- `test_data` (file 或 json string)

9. 验收标准
1. 软件能自动识别音频设备并正确配置通道。
2. 能在5秒内自动将耳机音量调节至94dB±0.5dB。
3. 能依次完成频响、失真、相位、冲击响应、动态线性偏离五项测试，并生成有效数据。
4. 产品信息录入和照片上传功能正常，必填项未填时禁止开始测试。
5. 测试数据能成功上传至指定网站，并返回成功状态。
6. 界面符合易用性要求，无崩溃或明显错误。

11. 附录
11.1 动态线性偏离计算方法详细说明
- 基准测试：在94dB SPL下测量频响曲线R94(f)。
- 高电平测试：提高播放增益10dB，测量频响曲线R104(f)（注意：需确保不削波，且耳机和麦克风不过载）。
- 低电平测试：降低播放增益10dB，测量频响曲线R84(f)。
- 偏离曲线：D(f) = R104(f) – R84(f) （单位dB）。
- 若D(f)接近0，说明线性度良好；若在某些频段偏离过大，表明存在非线性失真。
